# Contribution Report – CS 5542 Lab 6

Author: Adrija Ghosh  
Repository: LAB_4-APP  
Course: CS 5542 – Big Data Analytics and Applications

## Overview

This project was implemented entirely by a single contributor. Lab 6 builds upon the previously developed Lab 4 Retrieval-Augmented Generation (RAG) application by extending it with an AI agent layer capable of reasoning over user queries, selecting tools, retrieving evidence, and generating grounded responses. The goal of this extension was to transform the earlier RAG pipeline into an agent-based analytics system with tool-calling capability and an interactive user interface.

## Implementation Responsibilities

All system components were designed, implemented, tested, and documented by the author.

The Lab 4 application already included a basic retrieval pipeline and Streamlit interface. In Lab 6, the following enhancements were developed and integrated into the existing repository.

### Agent Architecture

An AI agent layer was implemented to interpret user queries and determine which tools should be executed to produce a response. The agent includes a system prompt, reasoning loop, tool-calling structure, and fallback execution mode when an external API key is not available.

File implemented:

lab6/agent.py


The agent processes user questions, selects appropriate tools, executes them, and generates grounded responses supported by retrieved evidence.

### Tool Development

Existing project components were converted into callable tools so that the agent could automatically execute them when needed. The tools include evidence retrieval, RAG query execution, and evaluation logging.

Files implemented:

lab6/tools.py
lab6/tool_schemas.py


Each tool includes clear docstrings, input/output type hints, and error handling to ensure that failures do not crash the application.

### Streamlit Application Integration

The existing Streamlit application from Lab 4 was extended to support agent interaction. A chat-based interface was implemented to allow users to submit queries and receive responses generated by the agent.

File implemented:

lab6/streamlit_app.py


The interface supports user input, conversation history, loading indicators during agent execution, and formatted responses containing evidence citations.

### Logging and Evaluation

A logging mechanism was implemented to track system behavior and evaluation metrics. The logs record query execution details such as retrieval mode, latency, returned evidence identifiers, and faithfulness checks.

Log files generated:

lab6/logs/agent_query_metrics.csv
lab6/logs/agent_execution_log.txt


These logs were used to analyze system performance and support the evaluation section of the lab report.

### Antigravity IDE Integration

Google Antigravity IDE was configured to analyze the repository and assist with code development. The IDE provided suggestions regarding project structure, tool configuration, and improvements to the agent implementation.

Deliverable created:

task1_antigravity_report.md


The report documents prompts used with Antigravity, suggested improvements, accepted changes, and reflections on the IDE's role as an intelligent assistant.

### Evaluation

Three evaluation scenarios were designed to test the agent system:

1. A simple query requiring a single tool call  
2. A medium query requiring retrieval and summarization  
3. A complex query requiring reasoning and synthesis of retrieved evidence  

The evaluation results were documented in:


task4_evaluation_report.md


Metrics such as latency, evidence usage, reasoning steps, and response accuracy were analyzed.
